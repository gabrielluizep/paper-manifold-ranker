\begin{abstract}
% Assessing the quality of telecommunications services across municipalities with heterogeneous structural, geographic, and socioeconomic conditions requires methodologies that go beyond simple weighted averages. This paper introduces a Graph Neural Network (GNN)-based manifold ranking framework that learns a latent representation of municipality-level telecom quality indicators and produces a unified ranking score. The proposed architecture employs Residual Graph Convolutional layers operating on a $k$-nearest-neighbor graph constructed from 18 infrastructure and quality-of-service indicators monitored by the Brazilian National Telecommunications Agency (ANATEL). The model is trained with a multi-objective loss function that simultaneously enforces pairwise ranking consistency, input reconstruction fidelity, geometric smoothness on the learned manifold, and directional alignment with domain knowledge. A monotonicity constraint is imposed on the scoring head to guarantee that improvements in underlying indicators always translate into higher scores. We validate the approach on a longitudinal dataset covering 4,088 Brazilian municipalities over 30 monthly periods (July 2022--December 2024), totaling over 82,000 municipality--period observations. Ablation studies across six experimental configurations demonstrate the contribution of each loss component. Temporal stability analyses via spaghetti plots reveal that the manifold ranker produces smoother and more consistent rankings than the regulatory baseline (IQS), while geographic visualizations confirm alignment with known infrastructure patterns. The framework provides a principled, data-driven alternative to expert-defined composite indices for regulatory quality assessment at national scale.
\end{abstract}

\section{Introduction}

Ranking municipalities in terms of telecommunications \gls{qos} poses a challenge to regulatory agencies. The ranking is useful not only for diagnostic purposes but for technical and economical evaluation of a given municipality or region. 

Defining a ranking strategy is not trivial. One of the problems raised by this challenge are the absence of stablished ground-truth on which cities are better or worse, therefore, evaluating models results depends on contextual
% requiring a field specialist for the analysis.
knowledge rather than objective comparison. 
Also, selecting the essential variables to make this analysis is not clear, not only \gls{qos} indicators are required to define the strategy but also covariates such as infrastructure and \gls{qoe} indicators. 

Currently, regulatories agencies uses linear composites to stablish \gls{qos} scores that are used to ranking municipalities. \gls{anatel} defined the \gls{iqs} through \gls{dvr}, that encapsulates \gls{qos} metrics in single valued score to define the telecommunications quality for each municipality.

This pointwise approach lacks a gobal overview of the performance on the municipalities.
% with similar performance characteristics.
By not considering a dominance from one over another entities, the ranking became faulty, with scores indistinguishable one from another. Similar entites are attributed near identical values, and the ocurrences are cluttered on narrow regions of the support domain.

% This is evidentiated when applying said pointwise linear strategies to real data
To improve distinguishability, one strategy that can be adopted is to compare features from two municipalities and define which one dominates over the other. To find these similarities it is possible to construct a graph, similar entites will be close to each other on the data space.

To asses all the presented challenges, we propose Manifold Ranker, a GNN-based Pareto-inspired model that learns the geometry of the space where the data live in, based on a constructed \gls{knn} graph, and by entity pairwise competition regresses a score value $s_i$ for each municipalities $i$.

\section{Dataset and Baseline Creation} \label{sec:dataset}

For the dataset construction we acquired \gls{qos} and \gls{qoe} indicators, and infrastructure covariates provided by \gls{anatel},
creating a matrix $\Xraw \in \mathbb{R}^{i \times d}$, where $i$ correspond to the index key and $d$ correspond to the number of features.
$\Xraw$ is indexed by two keys: \textit{month}, which identifies the observation period, 
and \textit{municipality}, the official \gls{ibge} numeric code that uniquely identifies each municipality. 
Each row therefore corresponds to one municipality--month pair.

As \gls{qos} features we selected eight features normalized to $[0,1]$ with higher values denoting superior performance,
$\mathrm{IND}_1$ (voice-call connection rate), 
$\mathrm{IND}_2$ (voice-call drop rate, complemented),
$\mathrm{IND}_3$ (data-connection rate),
$\mathrm{IND}_4$ (download and upload speed compliance rate),
$\mathrm{IND}_5$ (data-connection latency, complemented),
$\mathrm{IND}_6$ (jitter, complemented),
$\mathrm{IND}_7$ (packet-loss rate, complemented), and
$\mathrm{IND}_8$ (service availability rate). 
For \gls{qoe} feature, it was selected the $\mathrm{ISG}$ is the General Satisfaction Index, which is a composite of \gls{qoe} indicators for a given tuple municipality--period.
Infrastructure is represented on the dataset by $\mathrm{IBC}$, which is the Brazilian Coverage Index,
and the $\mathrm{INF4}_{\mathrm{UP},\{\mathrm{3G,4G,5G}\}}$ and $\mathrm{INF4}_{\mathrm{DL,\{\mathrm{3G,4G,5G}\}}}$ indicators, which records the median download speeds over each mobile data technology: 3G, 4G, and 5G.

To establish a baseline comparison with the currently applied metodologies in regulatory agencies, we present the consolidated calculated scores the legacy \gls{iqs} and the recently revised \gls{iqs}. Both of them linearly combines the same \gls{qos} features selected to use in the model.

\subsection{Legacy IQS}

The Anatel legacy composite, $\iqsleg$, aggregates the $K$ normalized
technical quality-of-service indicators through a fixed regulator-defined convex
combination,
\begin{equation}
    \iqsleg = y_{\max} \sum_{k=1}^{K} w_k^{\mathrm{leg}} \, \mathrm{IND}_k,
    \label{eq:qos_leg}
\end{equation}
where $y_{\max} = 10$ is the upper bound of the user satisfaction scale, each
$\mathrm{IND}_k \in [0,1]$ is a normalized technical indicator with higher values
denoting superior performance, and the weight vector $w^{\mathrm{leg}} \in \Delta^{K-1}$
lies on the probability simplex $\Delta^{K-1} = \{w \in \mathbb{R}^K : w_k \geq 0,\,
\sum_k w_k = 1\}$. Under the legacy specification, the weights are fixed at
$w^{\mathrm{leg}} = (0.1,\, 0.1,\, 0.1,\, 0.2,\, 0.1,\, 0.1,\, 0.1,\, 0.2)$,
assigning greater relevance to $\mathrm{IND}_4$ and $\mathrm{IND}_8$, while distributing equal
weight among the remaining indicators.


\subsection{Revised IQS}

The Anatel revised composite, $\iqsrev$, introduced in 2025 through
the updated \gls{dvr}~\cite{dvr2025}, replaces the fixed weights
with a performance-dependent scheme combined with a coverage bonus. The indicators are
first ranked from worst-performing to best-performing within each municipality, and the
weight assigned to rank position $r$ is
\begin{equation}
    \alpha_r = \frac{1}{r \, H_K},
    \label{eq:alpha_r}
\end{equation}
where $H_K = \sum_{j=1}^{K} 1/j$ is the $K$-th harmonic number. The revised composite
is then defined as
\begin{equation}
    \iqsrev = y_{\max} \sum_{r=1}^{K} \alpha_r \cdot \mathrm{IND}_{\pi(r)} + B_{\mathrm{cov}},
    \label{eq:qos_rev}
\end{equation}
where $\pi$ is the permutation ordering indicators from worst to best performance, and
$B_{\mathrm{cov}}$ is a coverage bonus corresponding to a fixed increment of five points,
added whenever coverage with 4G or higher technology in the municipality's urban area
reaches at least 95\%. When $B_{\mathrm{cov}} = 0$, the revised rule reduces to a linear
function of the indicators; when $B_{\mathrm{cov}} \neq 0$, it becomes affine. The
inverse-harmonic weighting assigns greater importance to the worst-performing indicators,
penalizing municipalities with more critical technical dimensions, in contrast to the
static weight distribution of the legacy composite.

\subsection{Preprocessing}

By exploring the selected data that consists the dataset, we verify that it is composed of heavy-tailed, heterogeneous data. Aiming to transform the data with that characteristics into a suitable representation for neural optimization we define a preprocessing step.

To reduce the dynamic range of the features that follows power-law or log-normal-like distributions, common among infrastructure data we apply a Log1p transformation, shown in Eq.~\ref{eq:log1p}, for numerical stability this transformations prevents the $\log(0)$ by adding one to the paramater.

\begin{equation}
    \Xlog = \log(1+\Xraw).
    \label{eq:log1p}
\end{equation}

To reduce outliers we apply a Robust scaler, which filter only the date within an interquantile range selected to be $\{0.05,0.095\}$, Eq.~\ref{eq:robust_scaling} shows the processing.

\begin{equation}
    \tilde{x}_{i,j} =
        \frac{\xlog_{i,j} - \mathrm{median}(\xlog_{i,j})}
        {Q_{0.95}(\xlog_{i,j})-Q_{0.05}(\xlog_{i,j})}.
    \label{eq:robust_scaling}
\end{equation}

From the robust-scaled features, we apply a Standar scaling, Eq.~\ref{eq:std_scaling} centering the variables into zero mean and unit variance, conditioning for first-orders optimizers, and making 

\begin{equation}
    \Xmodel_{i,j} = \frac{\tilde{x}_{i,j} - \mu_j}{\sigma_j},
    \label{eq:std_scaling}
\end{equation}

\noindent
where $\mu_j$ and $\sigma_j$ are the mean and the standard deviation for the $j$ feature across all municipalities.

Additionally from the $\Xmodel$, we create a proxy matrix $\Xpareto$ that serves as support for the model training serving as a ``compass'' general directional guide. $\Xpareto$ is constructed as shown in Eq.~\ref{eq:pareto}, by Min-Max scaling the $\Xlog$

\begin{equation}
    \Xpareto_{i,j} = 
        \frac{\xlog_{i,j} - \min_i \xlog_{i,j}}
        {\max_i \xlog_{i,j} - \min_i \xlog_{i,j}} 
        \in [0,1],
    \label{eq:pareto}
\end{equation}

Then, a scalar proxy utility is defined as the mean across features

\begin{equation}
    u_i = \frac{1}{D} \sum_j \Xpareto_{i,j},
\end{equation}

\noindent
this proxy is not a true Pareto ranking; it is a heuristic that proivdes a consistent direction for training in the abscence of labeled rankings.


\subsection{Graph Construction and Manifold Structure}

Graphs are mathematical structures capable extracting relationship from data. These relationship enables partial property sharing, having local properties. Geometrically we can visualize this as an estimation of a manifold.

We selected the \gls{knn} as the method for constructing the graph. This method calculates the distance between each node and select the $k$ nearest neighbors by using a distance metric, Minkowski in this case, as a result we get a index matrix with the nearest neighbors for each node $\mathcal{I} \in \{1,...,N\}^{N \times k}$, with $N$ being the number of nodes.

Most implementations of \gls{knn} algorithm traces the most nearest neighbor as the node itself, distance zero from itself, i.e., $i \rightarrow i$, creating a loop in the graph. This loop brings a residual glance to the model, that can be explored to prevent information vanishing.
